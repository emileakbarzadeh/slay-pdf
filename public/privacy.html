Privacy

Your PDFs stay in this browser.

Slay PDF is a local PDF editor. Documents and edits stay in this browser, and passwords are never saved.

No app-server uploads

Editing, previews and exports run in the browser instead of sending documents to a Slay PDF server.

Local workspace

Recent workspace data is stored in browser storage so work can continue after a refresh.

Passwords not saved

Export passwords are used for the export action only and are never saved by Slay PDF.

What stays local?

Imported PDF files, images, page edits, annotations, signatures, form edits, OCR work and exports are handled in the browser. The app is designed as a static site rather than a hosted document processing service.

What is stored?

Slay PDF stores the active workspace in browser storage so recent work can be restored. Using New PDF or clearing recent work removes that local workspace state from the app.

What about passwords?

Passwords for encrypted PDF exports are not saved. They are used only for the export currently being generated.

What should I still be aware of?

Your browser, operating system, extensions, backups, file downloads and network environment are outside Slay PDF's control. Use a trusted device and browser profile for sensitive documents.
