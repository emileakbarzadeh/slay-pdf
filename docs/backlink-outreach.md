# Backlink Outreach

Use this checklist when submitting Slay PDF to directories, roundups, newsletters, resource lists, or comparison pages.

## Primary URLs

- App: https://slaypdf.com/
- Backlink kit: https://slaypdf.com/link-to-slay-pdf.html
- Free PDF editor overview: https://slaypdf.com/free-pdf-editor.html
- Adobe Acrobat alternative: https://slaypdf.com/adobe-acrobat-alternative.html
- Source: https://github.com/emileakbarzadeh/slay-pdf

## Copy

Short:

> Slay PDF is a free local PDF editor and Adobe Acrobat alternative that runs in the browser, so documents stay on your device.

Directory:

> Slay PDF helps people split, merge, sign, resize, posterise, redact, OCR and edit PDFs locally in a modern browser. It is a static web app, so there is no app-server PDF upload for everyday edits.

Markdown:

```md
[Slay PDF](https://slaypdf.com/) - free local PDF editor for splitting, merging, signing and editing PDFs in the browser.
```

HTML:

```html
<a href="https://slaypdf.com/">Slay PDF - free local PDF editor</a>
```

## Good-Fit Targets

- Open-source app lists.
- Privacy and no-upload tool roundups.
- PDF utility directories.
- Adobe Acrobat alternative comparison pages.
- Student, teacher, nonprofit, freelancer, and small-business resource pages.
- Browser-based productivity tool directories.
- Developer launch posts that can link to both the app and the GitHub repo.

## Submission Notes

- Prefer natural anchor text: `Slay PDF`, `free local PDF editor`, `browser PDF editor`, `open-source PDF editor`, or `Adobe Acrobat alternative`.
- Link to the homepage for app directories and the Acrobat alternative page for comparison lists.
- Mention that the app is client-side and static, but avoid implying documents are protected from browser extensions, malware, or the user's own device.
- Include the GitHub repo where open-source proof helps the listing.
- Track submitted URLs, account used, date submitted, and status in a separate private sheet if credentials or private contact details are involved.
